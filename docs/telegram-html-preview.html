CoCo’s multi-machine model is deliberately hub-and-spoke: one active Telegram-facing controller, plus one or more non-Telegram agents reachable only over Tailscale. [controller_runtime.py](/Volumes/Data/Github/macOSAgentBot/coco/src/coco/controller_runtime.py) is a thin bootstrap: it logs config and starts Telegram polling. From the files you asked for, controller RPC startup is not done there, so that part is likely wired deeper in the bot/session layer.

[controller_rpc.py](/Volumes/Data/Github/macOSAgentBot/coco/src/coco/controller_rpc.py) defines the agent-to-controller callback surface: `controller/heartbeat`, `controller/notification`, and `controller/request`. That is how remote agents report liveness and forward app-server events back to the Telegram side.

[agent_rpc.py](/Volumes/Data/Github/macOSAgentBot/coco/src/coco/agent_rpc.py) defines the controller-to-agent control surface. It exposes machine ping/probe, browse, folder session discovery, thread listing, ensure/resume/fork/rollback, input sending, and document relay. [cluster_rpc.py](/Volumes/Data/Github/macOSAgentBot/coco/src/coco/cluster_rpc.py) is the transport: newline-delimited JSON over TCP, authenticated by a shared secret, with one request per connection and simple request/response semantics.

[node_registry.py](/Volumes/Data/Github/macOSAgentBot/coco/src/coco/node_registry.py) is the persistent cluster state store. It records machine identity, status, last-seen time, browse roots, RPC endpoint, and controller-related flags, then derives online/offline transitions from heartbeats plus timeout checks.

**Notable Design Tradeoffs**

The design is intentionally additive and conservative: remote support is a thin RPC layer on top of the existing local Codex session model, which helps migration safety and keeps single-machine installs working. The cost is that this is not a full distributed control plane: no automatic controller failover, no generic cross-machine job scheduler, no folder sync/handoff, and no terminal mirroring.

Security and transport are also pragmatic rather than heavy-duty. RPC trusts private Tailscale reachability plus a shared secret, which is simple and operationally cheap, but it is not a richer identity-bound protocol by itself. Similarly, liveness is more robust than pure heartbeats because of brokered peer probing, but still focused narrowly on node availability rather than broader cluster coordination.
